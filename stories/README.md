# Stories Backlog

Track each user story as a standalone markdown file in this folder so requirements live next to the code. Workflow:

1. Duplicate `stories/template.md` for each new story and update the metadata fields.
2. Keep every story focused on a single user-facing outcome with explicit acceptance criteria.
3. Break the work into 2–5 engineering tasks that each fit comfortably inside a day.
4. Link relevant code, env vars, or commands directly in the story so context stays close.
5. When a story is done, keep the file for historical context and mark tasks complete.

Suggested naming: `YYYY-MM-short-slug.md` (for example, `2026-03-trip-suggestions.md`).
